AI Pothole Detection System

Computer Vision Project

Developer :

Storm Schutte

Duration :

Municipal Project

Technology :

CNN, Deep Learning, Computer Vision

Accuracy :

98% Detection Rate

This project involved developing a computer vision pipeline to detect and count potholes on city roads in real time. The solution was designed for a municipal client who required an automated system to quantify road surface damage across different regions.

A forward-facing camera was mounted on vehicles to continuously capture road footage. The video stream was processed by a custom deep learning model that I architected and trained from scratch. The model was based on a convolutional neural network (CNN) optimized for object detection in challenging outdoor environments with variable lighting and road conditions.

The workflow included data acquisition and labeling of pothole images with manual annotation of bounding boxes, model development using a self-built CNN tailored for pothole detection with transfer learning techniques, and implementation of a video processing pipeline that ingests live footage, runs frame-by-frame detection, and outputs pothole counts with minimal latency.

The system achieved 98% detection accuracy with near-zero false positives, validated on unseen test data and field trials. The system proved highly effective in automating road condition monitoring, significantly reducing manual inspection requirements and enabling data-driven decision making for city infrastructure maintenance.

Multilingual Semantic Search for Automotive CRM Systems

Retrieval-Augmented Generation (RAG) Project

Role :

Team Lead

Duration :

Academic Research Project

Technology :

Weaviate â€¢ OpenAI â€¢ spaCy

Accuracy :

98% (vs 26% baseline)

As team lead, I designed and deployed a retrieval-augmented semantic search system for an automotive CRM platform, in collaboration with my professor, three developers, and government stakeholders. The system solved the challenge of delivering accurate, multilingual search results in a domain where traditional keyword search failed due to data diversity and ambiguity.

Problem Statement: The automotive industry dataset contained millions of records covering brands, models, parts, and regional variations. User queries were highly ambiguous ("Sonata 2.0 vs Sonata Hybrid," "SUV with 4-wheel drive"), and frequently cross-lingual (Korean/English). Traditional keyword search engines returned irrelevant results, with baseline accuracy ~26% on test queries.

Machine Learning & Data Science Approach: Query Understanding & Preprocessing - Built an NLP pipeline with spaCy and custom regex/entity extractors. Normalized queries, extracted attributes (make, model, feature), and reduced noise for embedding generation. Embedding Models Evaluation - Benchmarked open-source embeddings (e.g., Sentence-BERT, multilingual MiniLM) against proprietary models. Found OpenAI's large embedding model superior in handling multilinguality, eliminating the need for multiple per-language embeddings. Conducted quantitative evaluation with retrieval precision/recall and semantic similarity scoring.

Vector Search Integration: Implemented Weaviate as the vector database for high-dimensional nearest-neighbor search. Queries were transformed into embeddings, compared against vectorized car records, and ranked by semantic closeness. Added re-ranking heuristics to penalize near-duplicate results and improve diversity. Explored Neo4j for hierarchical relationships but excluded due to complexity vs. marginal benefit.

System Architecture: Pipeline - Query â†’ NLP preprocessing â†’ Embedding model â†’ Vector DB search (Weaviate) â†’ Result re-ranking. Deployment - Containerized services, with embeddings served via API and Weaviate deployed in a cloud cluster. Monitoring - Set up logging and evaluation dashboards to track query success rates and embedding drift over time.

Results & Impact: Increased search accuracy from 26% to 98%, validated against a labeled benchmark of real CRM queries. Delivered multilingual semantic search without additional translation pipelines, lowering system complexity. Supported government-backed initiatives in Korea, providing scalable search infrastructure for automotive data management. Demonstrated that retrieval-augmented search (RAG) techniques, powered by embeddings, can outperform keyword search in highly diverse industry datasets.

Key Learnings: Embeddings generalize across languages better than fine-tuned monolingual models when the goal is scalable retrieval. Vector databases (Weaviate) are production-ready solutions for semantic CRM systems at industry scale. While graph databases add conceptual richness, the trade-off must be weighed against maintainability and latency.
